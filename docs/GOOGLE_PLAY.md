# Google Play: листинг, Data safety, политика, рейтинг, скриншоты

Материалы для заполнения [Google Play Console](https://play.google.com/console) — правьте под фактическое поведение приложения и юридическую позицию.

## Требования Play к target API

Перед публикацией сверяйтесь с [официальными требованиями Google Play](https://developer.android.com/google/play/requirements/target-sdk) к целевому API — минимальный допустимый уровень периодически повышается.

В проекте в Gradle: **`compileSdk = 35`** и **`targetSdk = 35` (Android 15)** — целевой уровень совпадает с уровнем компиляции и соответствует ожиданиям консоли для публикации (при необходимости сверяйте актуальный порог на странице требований выше).

Низкий `targetSdk` (например 29) для обычной публикации в Play **не подходит**; такой вариант оставляют только для особых сценариев вне магазина.

## Data safety (форма «Безопасность данных»)

Заполняйте по **фактическому** сбору и передаче данных. Ниже — типичные пункты для клиента Rocket.Chat + FCM; отметьте только то, что применимо.

- **Сетевые запросы**: подключение к серверу Rocket.Chat, указанному пользователем (URL, учётные данные, содержимое чатов на стороне сервера и при необходимости в TLS).
- **Учётные данные**: логин/пароль или OAuth — передача на сервер пользователя; локально — сессия в приложении (DataStore/Room по реализации).
- **Push (FCM)**: токен устройства регистрируется на сервере Rocket.Chat для доставки уведомлений; в консоли Firebase — настройки проекта по политике Google.
- **Камера / микрофон**: для звонков (Jitsi) — по запросу пользователя, в контексте звонка.
- **Уведомления**: отображение входящих сообщений и вызовов — по политике ОС и разрешениям.
- **Резервное копирование Android** (`allowBackup`): если включено — укажите, что данные могут участвовать в резервном копировании ОС (уточните по манифесту).

Шифрование: для трафика к серверу по HTTPS опишите это; при использовании HTTP к своим серверам — честно укажите модель (см. cleartext в приложении).

## Политика конфиденциальности (URL)

В консоли указывается **публичный HTTPS URL**. Подготовьте страницу (сайт компании / GitHub Pages / хостинг политики), где описано:

- какие данные обрабатывает приложение (сеть, аккаунт, push, опционально звонки);
- что данные обрабатываются сервером Rocket.Chat пользователя, а не вами как разработчиком, если так и есть;
- контакт для запросов пользователей.

Черновик структуры: см. [PRIVACY_POLICY_TEMPLATE.md](PRIVACY_POLICY_TEMPLATE.md) — адаптируйте под юрисдикцию и бренд.

| Тип | Требования (проверьте актуальную справку Play) |

| 7" / 10" планшет | Если заявляете поддержку планшета. |
| Wear / TV | Только если распространяете на эти форм-факторы. |

Снимки: экран входа, список комнат, чат, уведомления (по желанию), настройки — без чужих персональных данных и без выдуманных функций.
